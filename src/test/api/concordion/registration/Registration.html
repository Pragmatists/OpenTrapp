Registration

Background:

  • Given user Homer already exists

Example 1: Creating new Work Log Entry

  • Given client has been authenticated as Homer
  • When POST request is being send to: /endpoints/v1/employee/Homer/work-log/entries
  • And request body contains following data: 
    {
    "workload": "1h 30m",
    "projectName": "ApiDesign",
    "day": "2014/01/01"
}
  • Then response status should be equal to HTTP/1.1 201 Created
  • And following work log entries should exist:
    Project Workload Employee
    ApiDesign 1h 30m Homer

Example 2: Creating new Work Log Entry (for given day)

  • Given client has been authenticated as Homer
  • When POST request is being send to: /endpoints/v1/employee/Homer/work-log/entries
  • And request body contains following data: 
    {
    "workload": "1h 30m",
    "projectName": "ApiDesign",
    "day": "2013/12/31"
}
  • Then response status should be equal to HTTP/1.1 201 Created
  • And following work log entries should exist:
    Day
    2013/12/31

Example 3: Response of successful request

  • Given client has been authenticated as Homer
  • When POST request is being send to: /endpoints/v1/employee/Homer/work-log/entries
  • And request body contains following data: 
    {
    "workload": "1h 30m",
    "projectName": "ApiDesign",
    "day": "2014/01/01"
}
  • Then response status should be equal to HTTP/1.1 201 Created

Example 4: Invalid Work Log Expression

  • When POST request is being send to: /endpoints/v1/employee/Homer/work-log/entries
  • And request body contains following data: 
    {
    "workload": "1h 30m"
}
  • Then response status should be equal to HTTP/1.1 400 Bad Request

Example 5: Invalid Content Type

  • When POST request is being send to: /endpoints/v1/employee/Homer/work-log/entries
  • And content type is set to text/html
  • And request body contains following data: 
    {
    "workload": "1h 30m",
    "projectName": "ApiDesign",
    "day": "2014/01/01"
}
  • Then response status should be equal to HTTP/1.1 405 Request method 'POST' not supported

Example 6: Missing Employee

  • When POST request is being send to: /endpoints/v1/employee/Unknown/work-log/entries
  • And request body contains following data: 
    {
    "workload": "1h 30m",
    "projectName": "ApiDesign",
    "day": "2014/01/01"
}
  • Then response status should be equal to HTTP/1.1 404 Not Found
